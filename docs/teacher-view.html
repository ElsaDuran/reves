Reves' project memo

What exactly is Reves?

Reves is a Machine Learning project based on predicting movie revenue from the basic features of a movie, as genre, cast, crew, release date, budget, etc. It is a business tool mainly aimed for film producers, who want to know how much money the movie is going to make before even producing it.

Where does data come from?

This is a basic model developed using The Movies Dataset, a public project in Kaggle. Of course, the accuracy of our model is conditioned to the limited ammount of data we have in this public dataset.

You can check the Kaggle website here.

Why did we decide to develope this specific project?

We wanted to create a project that would actually make sense in real life. We didn’t want to spend hours of work just for a research project to store in a drawer. The movie topic was chosen just because we liked it, and the idea of making a data-based business tool came to us analyzing the data we had, and thinking how could we make money out of it.

Why is it relevant as a Final Project?

During the developing of Reves, we have had to face the different challenges that have been raised as such from the teachers during this course: exhaustive data cleaning,choosing relevant variables, identifying and applying the most convenient ML model, visualizing the data, and, as a bonus track, using data in a real production enviroment by developing a webapp front and backend.

We have also tried to apply Deep Learning to experience this technology and to try to better the accuracy, despite the loss of information about how the model actually works. Unfortunately, we couldn't get good results by now, so we are leaving this type of analysis as a next step for Reves 2.0.

Which technologies have we used?

  • Python for data cleaning, engineering and modelling.
  • Tableau for data exploration.
  • HTML & CSS for web front-end.
  • Flask and Python for webapp back-end.
  • Heroku for webapp hosting and publishing.

Why would it relevant for the industry?

Although the accuracy of our model is conditioned to the limited ammount of data we have in this public dataset, Reves hasn’t work its magic yet. It has a huge low-cost-growth-potential that will make it extremely attractive for film producing companies. The idea is simple: It is way more cheaper to collect the basic data from Internet for every movie ever made thank producing a single movie. Productions companies would only have to invest some money in harvesting the data and then apply the model (with some restructuring). Voilá: profit margins will rise.

Let's get started

Download data

If you haven't done so yet, you should clone or download our Github repo. Within this files, you will find:

  • 4 Jupyter Notebooks.
  • Python file with Reves' function library. You will import it at the beggining of each notebook, but you can review its content here.
  • Python files with Reves' app (called revesapp.py) and form (form.py)
  • Procfile and requirements.txt file for app's deployment
  • Pkl file with trained model
  • Docs and Static directories with HTML, CSS and static files for website.
  • Developing directory with former notebooks.
  • Tableau_viz directory with some plots for data exploration.

What do you need to run the project?

You don't need anything to run and use Reves' app from its website, but if you want to download and run Reves on your computer, you will need:

  • Python 3
  • Flask. Use pip install Flask
  • Unidecode. Use pip install unidecode
  • XGBoost. Use pip install xgboost
  • WTforms. Use pip install wtforms
  • IMDB extra file. Too large to upload to Github. You can download it from this link. Save it in your reves/data/imdb directory.

In order to run Notebook 3 (Modeling) in local:

  • Graphviz. Use pip install graphviz
  • Shap. Usepip install shap

    • To run Reves' web app in local, you'll need to go to the app's directory and run in your command prompt "python revesapp.py". Then, open the route given in your browser (Chrome if possible). If you just open .html files, CSS styles won't be displayed and apps' functionality won't be available.

    If you run Reves from the Internet, the first time you access the web it might take a wile to load the page, since we are using a Heroku's free account and dynos sleep after 30 minutes of inactivity. Please, be patient.
