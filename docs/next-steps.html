Reves basics

As you already should know, Reves is a final project developed for our Master in Data Science degree. It's a basic model developed using The Movies Dataset, a public Project in Kaggle. Of course, the accuracy of our model is conditioned to the limited ammount of data we have in this public dataset and its reliability.

At the same time, you ought to know that we had limited time for building the project, and that is why there are some improvements yet to come.

Technical improvements

  • Responsive design:
    If you download the app in local, you can check that it is full responsive. Unfortunately, the app deployed in Heroku needs some adjustments for responsive design.

  • Autocomplete fields in form:
    In this initial version of Reves, the user has to check the correct spelling of those names in a different page called Index. To better user's experience using the form, we will add autocomplete fields for features like directors, scriptwriters, etc.

  • Database:
    We will add a SQL or SQLlite database to keep track of all the inputs collected by the app's form. We will use this info for trend analysis, know what the production companies are seeking and how they interact with our app, and also to be able to add a export in Excel button, so the company can have all their different iterations with the form store in one file to analyze and visualize the options.

Content & model improvements

  • Overview's NLP Analysis:
    On the initial dataframe we had an Overview column with a text that talks about the plot of the movie. It would be very interesting to analyize this text with NLP techniques to add extra info about the movie's plot to the model

  • Lower model's error:
    If you use a new or fake name in directors, cast names or other typelike features, the final predicted number is still pretty high. This is why, for unknown values, Reves applies an average score. We hope that in future versions of Reves, when we have more than 5K rows in our dataframe, we can change this parameter and allow the model to learn better to better data and minimize this error.

  • Complement model with social listening:
    We think that it could be pretty cool and interesting to complement the information given to Reves with a social listening factor/score based on similar movies. Sometimes, prior to the movie's releasing, there's a social speech that can affect the final movie's box office. If people are enthusiastic about the movie and talk about it at social platforms, there's more probability of more audience.

  • Use producers data:
    As explained in notebook 1, we have found extra databases with info about directors and scriptwirters to merge with the initial dataframe. That's why we have been able to use this features in our model. However, we couldn't find a dataset with producers info, thought we think it will add significant value to our model.

  • Analyze First Weekend Box Office:
    Experts say that the final revenue is extremely related to the movie's first weekend box office. It will be great tho have this data available to check if it's actually true and, in this case, use is as an input for the model.

  • Improve CPI-revenue transformation
  • Add estimated total budget and/or estimated marketing budget variables
