How to use Reves

Reves is a interactive web app where a production company can enter the main features of the movie they are about to produce and instantly know the potential revenue of the movie. As it is an interactive tool, you can play with: change the values, sumbit them and find the perfect featuring for your movie.

Where does data come from?

This is a basic model developed using The Movies Dataset, a public project in Kaggle. Of course, the accuracy of our model is conditioned to the limited ammount of data we have in this public dataset.

You can check the Kaggle website here.

How accurate is the result?

After final training, out model has proven to be very accurate if we take to consideration that we only have about 5K records to train the final model with 17 variables. That means, in terms of ML, that Reves has learned just about 4K in Train & Test evaluations. In addition to this, some of the columns values are a final score obtained by the processing of even more possible values, so we can really say that we have a way too small ammount of data because the final model has to actually learn from way more than 17 features.

Reves' evaluation scores:

  • Mean absolute error: 29715379.95372
  • Explained Variance Score: 0.9507864605328741
  • Xgbo score: 0.9545966853070289
  • Cross val score: 0.7796376923052251

There's an important handicap we should take account of: because the value to predict is that high (we are talking about several million dollars), the mean error is, therefore, pretty high too. During tests, we found out that error was lower and metrics higher if we tried to predict revenue without applying an CPI recalc. That's because values to predict would be lower, but they would also be less realistic if we trained the model this way. After Train & Test evaluation, we have retrained our model with the full dataset, so it might have learned slightly better, but there's still a lot of work to do with Reves.

If you want to learn more about our model, accuracy and feature relevance, go to our modeling notebook.

What happens when data is not in Training?

There are millions of actors, directors, producers and almost any of the input features that Reves uses to calculate potential revenue. Reves learns from the past, so if you have a rookie actor, or you are a new production company, your final result won't be very accurate since Reves has no data from the past to learn from. As Reves evolves, it will be trained with more and better data and the final result will be more reliable.

For now, when you enter some "new data" into de model, we will apply a mean-based value so the lack of experience won't affect the final result, not in a positive nor in a negative way. If you wish to learn more about Reves developing process, go to our Specifications section.

Run Reves on your computer

Download data

If you haven't done so yet, you should clone or download our Github repo. Within this files, you will find:

  • 4 Jupyter Notebooks. More about Reves' structure here.
  • Python file with Reves' function library. You will import it at the beggining of each notebook, but you can review its content here.
  • Python files with Reves' app (called revesapp.py) and form (form.py)
  • Procfile and requirements.txt file for app's deployment
  • Pkl file with trained model
  • Docs and Static directories with HTML, CSS and static files for website.
  • Developing directory with former notebooks.
  • Tableau_viz directory with some plots for data exploration.

What do you need to run the project?

You don't need anything to run and use Reves' app from its website, but if you want to download and run Reves on your computer, you will need:

  • Python 3
  • Flask. Use pip install Flask
  • Unidecode. Use pip install unidecode
  • XGBoost. Use pip install xgboost
  • WTforms. Use pip install wtforms
  • IMDB extra file. Too large to upload to Github. You can download it from this link. Save it in your reves/data/imdb directory.

In order to run Notebook 3 (Modeling) in local:

  • Graphviz. Use pip install graphviz
  • Shap. Usepip install shap

    • To run Reves' web app in local, you'll need to go to the app's directory and run in your command prompt "python revesapp.py". Then, open the route given in your browser (Chrome if possible). If you just open .html files, CSS styles won't be displayed and apps' functionality won't be available.

    If you run Reves from the Internet, the first time you access the web it might take a wile to load the page, since we are using a Heroku's free account and dynos sleep after 30 minutes of inactivity. Please, be patient.
